# Auto-Build Setup for RockOut

## Quick Start

### Option 1: Use the Auto-Build Script (Recommended)

Run the auto-build watcher in a terminal:

```bash
./auto_build.sh
```

This will:
- Watch for changes to Swift files in the `Rockout/` directory
- Automatically rebuild when files are modified
- Show build status (success/failure)
- Continue watching until you press Ctrl+C

### Option 2: Enable Xcode's Automatic Building

1. Open Xcode
2. Go to **Xcode → Settings → Locations**
3. Enable **"Automatically Refresh Views"** (if available)
4. Or use **Product → Build** (⌘B) manually

### Option 3: Install fswatch for Better Performance

```bash
brew install fswatch
```

Then run:
```bash
./auto_build.sh
```

fswatch provides faster, more efficient file watching compared to polling.

## Manual Build Commands

If you prefer to build manually:

```bash
# Clean build
xcodebuild -project Rockout.xcodeproj -scheme Rockout \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
  clean build

# Just build (no clean)
xcodebuild -project Rockout.xcodeproj -scheme Rockout \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
  build
```

## Tips

- Keep the auto-build script running in a separate terminal window
- The script will show build errors immediately
- Press Ctrl+C to stop the watcher
- Builds are incremental (only changed files are recompiled)
